# Project Overview

## Summary

This project is a ground-up HTTP/1.1 server written in C++20 using POSIX sockets on Linux. Its purpose is to build a real understanding of how a server accepts connections, reads request data, interprets HTTP messages, routes requests, and returns valid responses without relying on higher-level HTTP frameworks.

## Motivation

The project exists to strengthen low-level systems understanding while improving modern C++ design and implementation skills. Rather than using existing frameworks, the goal is to work directly with the mechanics that those frameworks normally hide, including networking, protocol handling, request flow, and server structure.

## Problem Statement

Modern web development often makes it easy to use HTTP without understanding how an HTTP server actually works internally. That creates a gap between using server technology and understanding the mechanics behind it. This project addresses that gap by making HTTP server implementation itself the learning target.

## Intended Audience

The primary audience is the project author. Secondary audiences include future employers, reviewers, or collaborators who may use the project to evaluate engineering depth, design discipline, and low-level systems understanding.

## Project Value

If successful, this project will produce a stronger understanding of HTTP server architecture, deeper practical C++ experience, and a better foundation for future backend, networking, and systems work. It also creates a concrete project artifact that demonstrates structured engineering from first principles.

## Why This Project Is Being Done Now

This project is being done now to strengthen core engineering fundamentals while refreshing and improving C++ skills. It also supports broader goals around building deeper systems knowledge before moving into more advanced networking, backend, and infrastructure work.
